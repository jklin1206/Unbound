# Skill Dossier Audit

Date: 2026-05-17

Scope: first pass after the muscle-up dossier. This checks whether the skill has usable training options, objective rank gates, exercise-library coverage, and a clear visual/dossier path.

## Priority Findings

| Skill | Tracking state | Training options | Library gaps | Dossier / asset next step |
| --- | --- | --- | --- | --- |
| `pp.pullup` | Gates are objective and clear, but the rank ladder currently starts at 1 pullup and labels that as Initiate. That may make the first real pull-up feel too low-value compared with the muscle-up model. | Good breadth: hangs, scap pulls, rows, negatives, band assist, AMRAP, tempo work. | Exercise catalog has the core names. The visible library has `Pull Up`, but not explicit entries for `Negative Pull-Up`, `Band-Assisted Pull-Up`, `Scapular Pulls`, or `Australian Row`. | Strong next candidate for a full dossier. Needs 4-panel visual: hang/pack, pull path, chin-over-bar, controlled descent. |
| `pp.strict-pullup` | Skill gates use `strict pullup`; plan uses `Strict Pull-Up`. This should normalize, but strict pull-up is not in the exercise catalog or visible exercise library, so logging can be awkward. | Good strength-focused plan: tempo, pause reps, bent-arm hang, loaded hang. | Add `strict pullup`, `pause pullup`, `bent-arm hang`, and maybe `loaded hang`. | Dossier should define strict standard: dead hang start, no kip, chin clears bar, controlled lower. |
| `cal.pushup` | Gates are objective and progressive. Comment says 10 reps = Forged, but code has Forged at 15 reps. | `cal.5-dips` | Gates are objective and clean, all based on `dip`. | Good options: bench dip, negative, band assist, support hold, tempo. | Visible library only has `Dips (Tricep)`, while gates expect `dip`; catalog has `dip` but not bench/negative/banded/tempo/support variants. | Dossier should split parallel-bar dip from straight-bar dip, since muscle-up now depends on straight-bar dip. |
| `cal.ring-dip` | Gates are objective and use `ring dip`, but ring dip is not in catalog or visible library. | `cl.hollow-body-30` | Rank gates use `hollow body hold`, but catalog uses `hollow hold`, and training plan uses `Hollow Rocks` / `Hollow + Arch Superset`. | `hs.wall-handstand-30` | Lower gates use `wall handstand`, upper gates require `handstand pushup`. | `hs.freestanding-hs-30` | Gates use freestanding handstand plus HSPU/press variants. That makes sense for elite tiers, but early progression may be too strength-gated instead of balance-gated. | Good balance plan: wall base, kick-up practice, shoulder taps, heel pulls. | Catalog/visible library lacks most of the actual drills: freestanding handstand, kick-up, heel pull, shoulder tap, tuck handstand. | Needs a multi-part visual guide with balance correction arrows and fall/safety cues. |

## Immediate Recommendations

1. Use the muscle-up structure as the template: objective rank gates, several training options, then a visual form tab.
2. Add missing exercise-library items before authoring too many more dossiers, otherwise the rank path will recommend movements users cannot cleanly log.
3. Normalize names across tier gates, catalog, visible library, and training plans. The first urgent mismatches are `hollow body hold` vs `hollow hold`, `handstand pushup` vs `wall handstand pushup`, and ring-dip variants missing from the library.
4. For hold skills, decide whether the app should track seconds directly. Until then, use variant gates carefully and pair them with objective strength proxies only where that proxy actually matches the skill.
5. Next dossier batch should be `pp.pullup`, `cal.pushup`, `cal.5-dips`, `cal.ring-dip`, and `cl.hollow-body-30`. Those cover the most reusable foundations for the rest of the tree.
