UNBOUND Product Architecture

The app needs one workout spine.

A user can follow today’s program, train a skill on the spot, add skill work into the plan, run a routine, or build their own workout. All of those should feel different at the surface, but resolve into the same planning and logging system underneath.

All Entry Points

Total ways a user can work out

Program

Follow today’s generated workout

Structured strength/hypertrophy day from TrainingProgram → ProgramDay → Workout. Saves as WorkoutLog.

Skills

Start a guided skill session

From skill detail or Program tab active goals. Uses SkillSessionView. Saves as SessionLog.

Skills

Quick-log a skill set

Fast reps/hold/load/RPE capture. Also saves as SessionLog, not WorkoutLog.

Program + Skills

Add skill to “Today’s Training”

Active goal scheduler routes skills into Program tab, but currently as separate session cards, not as blocks inside the main workout.

Custom

Create a custom workout

Custom exercises exist, but a full custom workout builder/launcher is not clearly unified as a first-class flow.

Routines

Run routines/challenges

Mobility, recovery, challenge flows use routine history, not the main workout log spine.

End State

Every path should create a Workout Ready Card.

The user should always land on a clear session object: today’s planned training, edited training, skill-only training, routine training, or custom training. The app can call them different things, but internally they should all become a single session draft.

Skill Integration

“Add to Program” should mean more than pinning a card.

Current

Active Goal

Skill is pinned into Program tab’s “Today’s Training” via active goals. It launches a separate skill session.

Target

Program Block

Skill becomes a scheduled block inside the day: warmup, primary skill, accessory skill, cooldown, or optional extra.

On the spot

User taps Train from skill detail. App creates a skill-only session draft.

Planned

User adds the skill to weekly plan. Scheduler places it on matching day types.

Embedded

User edits today’s workout and inserts handstand work before pressing work.

User Flow

What the user should experience

1

Open Program

Today has one clear workout ready, with strength + skill blocks already blended when appropriate.

2

Adjust

Add skill, swap exercise, shorten session, add custom movement, or save template.

3

Train

Same logger handles reps, holds, attempts, distance/time, load, sides, and RPE.

4

Resolve

One completion payload updates rank, XP, badges, progression, body attributes, coach memory, and history.

Missing Components

What needs to be incorporated

Unified session draft model

A planning object that can represent program, skill, custom, cardio, carry, routine, or mixed workouts before they are logged.

Unified performance log

A completion object richer than WorkoutLog: reps, load, hold seconds, side, distance, duration, calories, attempts, quality.

Workout builder

User-created workouts need first-class build, edit, save, and start flows.

Skill block insertion

Skill training should be insertable into today’s workout and schedulable into future days.

Scheduler rules

Rules for when skill work is warmup, primary, accessory, or separate session.

One completion pipeline

Every completed session should feed the same reward, rank, XP, attributes, progression, and coach-context systems.

Decision

The product rule

Anything that feels like training must become a session draft. Anything completed must become a unified performance log. Program, skill, custom, routine, and cardio are entry points, not separate progress systems.

Surface can vary

Skill sessions can feel skill-specific. Program days can feel like workouts. Routines can feel guided.

Data should converge

Underneath, all paths should resolve into shared blocks, metrics, and completion events.

Progress should be universal

No matter where the user trains, the app should understand what improved.
